Git Branching Strategy

Nifty Trading System — personal dev branch workflow with daily sync

1 Setup — run once

Create the dev branch on GitHub from main, then track it locally.

# On GitHub UI: create branch "dev" from main
git fetch origin
git checkout -b dev origin/dev
2 Daily sync

Every morning — pull any overnight commits from origin/main into your local dev and push to keep origin/dev current.

git fetch origin
git rebase origin/main
git push origin dev --force-with-lease

Skip git checkout dev if you're already on dev (you will be if you ended last session correctly).

3 Make changes

Branch off dev for each task. Never commit directly to dev or main.

git checkout -b feature/my-change
# ... edit files ...
git add -A
git commit -m "describe what and why"
git push origin feature/my-change
gh  pr create --base main
4 Raise PR → merge

Review and merge the PR on GitHub UI. PR targets main, not dev.

# GitHub UI: review → Merge PR #N → Confirm
5 Post-merge sync

Your merged commit is now on origin/main. Rebase dev to pick it up, push to sync origin/dev, then clean up the feature branch.

git fetch origin
git checkout dev
git rebase origin/main
git push origin dev --force-with-lease
git branch -d feature/my-change
git push origin --delete feature/my-change

Why push in Phase 5? The PR merge added a new commit to origin/main after Phase 2's push. After rebase, local dev is 1 commit ahead of origin/dev — so a push is needed to close that gap.
